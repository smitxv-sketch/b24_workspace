# 01 — Переменные и PHP в бизнес-процессах

---

## §1.1 Запись в переменную БП

### ✅ Единственный рабочий способ — `SetVariable`

```php
$this->SetVariable('имя_переменной', $значение);
```

```php
// Строка
$this->SetVariable('folder_url', 'https://example.com/folder/');

// Число
$this->SetVariable('total_count', 42);

// Результат вычисления
$sum = $a + $b;
$this->SetVariable('sum_result', $sum);

// Несколько переменных сразу
$this->SetVariable('status', 'OK');
$this->SetVariable('error', '');
```

### ✅ Альтернатива — `return` (только одна переменная)

```php
return $result;
// В настройках действия «PHP-код» → «Сохранить результат в переменную: имя»
```

### ❌ НЕ РАБОТАЕТ

```php
{=Variable:folder_url} = $result;   // синтаксис {=...} — только для ЧТЕНИЯ
$folder_url = $result;              // обычная PHP-переменная, БП её не видит
```

## §1.2 Чтение значений в PHP-коде

```php
// Числовое поле документа — без кавычек
$dealId = {=Document:ID};
$responsibleId = {=Document:ASSIGNED_BY_ID};
$folderId = {=Document:UF_CRM_FOLDER_ID};

// Строковое поле — в двойных кавычках
$title   = "{=Document:TITLE}";
$comment = "{=Variable:user_comment}";

// Переменная БП — в двойных кавычках
$existing = "{=Variable:some_var}";

// Использование в вычислениях
$newVal = $existing + 100;
$this->SetVariable('new_variable', $newVal);
```

**Правило:** числа — без кавычек, строки — в двойных кавычках.

---

## §1.3 Типы данных

| Тип переменной БП | Что передавать в SetVariable |
|---|---|
| Строка | Строку PHP |
| Целое число | `intval($x)` или число |
| Число с точкой | `floatval($x)` или число |
| Дата | `'09.12.2025'` (формат портала) |
| Да/Нет | `'Y'` или `'N'` |
| Список (enum) | ID значения списка (число) |
| Привязка к пользователю | ID пользователя (int) |
| Массив / объект | `json_encode($array, JSON_UNESCAPED_UNICODE)` |

### Передача массива между шагами

```php
// Шаг 1: запаковать
$files = [101, 202, 303];
$this->SetVariable('file_ids_json', json_encode($files));

// Шаг 2: распаковать
$files = json_decode("{=Variable:file_ids_json}", true);
foreach ($files as $fileId) { /* ... */ }
```

## §1.4 Обработка ошибок — минимальный шаблон

```php
<?php
try {
    CModule::IncludeModule('disk');

    $folderId = {=Document:UF_CRM_FOLDER_ID};
    if (empty($folderId)) {
        $this->SetVariable('status', 'ERROR: поле ID пустое');
        return;
    }

    $folder = \Bitrix\Disk\Folder::getById($folderId);
    if (!$folder) {
        $this->SetVariable('status', 'ERROR: папка не найдена');
        return;
    }

    // ... ваша логика ...

    $this->SetVariable('status', 'OK');

} catch (Exception $e) {
    $this->SetVariable('status', 'ERROR: ' . $e->getMessage());
}
```

## §1.5 Частые ошибки синтаксиса

```php
// ❌ Переменная БП в строке без кавычек
$text = '{=Variable:name}';    // не подставится — это строка

// ✅ В двойных кавычках
$text = "{=Variable:name}";    // подставится значение

// ❌ Массив напрямую
$this->SetVariable('data', [1, 2, 3]);   // может не работать

// ✅ Через JSON
$this->SetVariable('data', json_encode([1, 2, 3]));

// ❌ Забытая точка с запятой
$a = 1
$b = 2;  // Parse error

// ❌ Кавычки в строках
$text = "Он сказал "привет"";   // Parse error

// ✅
$text = "Он сказал \"привет\"";
$text = 'Он сказал "привет"';
```

## §1.6 Логирование для отладки

```php
// Вариант 1: в переменную БП (видно в интерфейсе)
$log = "folderId={$folderId}, status=OK";
$this->SetVariable('debug_log', $log);

// Вариант 2: в файл на сервере
file_put_contents(
    $_SERVER['DOCUMENT_ROOT'] . '/local/logs/bp_debug.txt',
    date('Y-m-d H:i:s') . ' — ' . print_r($data, true) . "\n",
    FILE_APPEND
);

// Вариант 3: через WriteToTrackingService (видно в логе БП)
$root = $this->GetRootActivity();
$root->WriteToTrackingService('Мой отладочный текст');
```
