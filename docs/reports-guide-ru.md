# Workspace Reports — руководство (RU)

Документ описывает текущую систему отчётов в Workspace:
- какие есть представления,
- какие фильтры применяются,
- что выводится на экране,
- какую управленческую задачу решает каждый режим.

---

## 1) Задача системы отчётов

Цель отчётов — дать руководителю быстрый ответ на 3 вопроса:

1. **Кто сейчас загружен / недозагружен?**
2. **Что уже сделано за период и где узкие места?**
3. **Как выглядит плановая занятость по дням/неделям/месяцам?**

Для этого в системе есть несколько представлений поверх одной и той же фильтрации.

---

## 2) Единая модель фильтрации

Все представления используют общий набор фильтров (левая панель):

- **Подразделение** (поиск + выбор)
- **Сотрудник**
- **Проект**
- **5 дат-фильтров**:
  - Дата создания
  - Плановая дата начала
  - Плановая дата завершения
  - Крайний срок
  - Фактическая дата завершения
- **Включить подчинённые отделы**
- **Сортировка**
- **Статусные срезы**:
  - Плановые (ждут выполнения)
  - В работе (выполняется/контроль)
  - Выполненные (завершены)
- **Группировка**:
  - Сотрудники
  - Отделы
  - Проекты

### Быстрый ввод дат

Для каждой из 5 дат есть кнопка **«Быстрый ввод»** с пресетами:
- Текущая неделя/месяц/квартал/год
- Последние 7 дней/месяц/квартал
- Ближайшие 7/30 дней, ближайший квартал/год

Фильтр считается активным, если заполнено хотя бы одно поле `с`/`по`.

---

## 3) Представление «Трудозатраты»

### Что решает

- Контроль фактических часов по сотрудникам/проектам/отделам.
- Понимание, куда ушло время за выбранный период.
- Поиск перерасхода по задачам (через `Δ`).

### Что выводится по задаче

- Заголовок задачи (`#ID + имя`)
- Статус
- Мета-строка:
  - `Дата`
  - `Срок`
  - `План`
  - `Факт`
  - `Δ`

### KPI (сверху)

- Всего часов
- ФОТ (оценка)
- Сотрудников
- Среднее на сотрудника

---

## 4) Представление «Занятость»

### Что решает

- Контроль календарной и сроковой дисциплины задач.
- Быстрый просмотр плановых дат по задачам в разрезе сотрудников/проектов.

### Что выводится по задаче

- Заголовок задачи (`#ID + имя`)
- Статус
- Мета-строка:
  - `Создана`
  - `Начало`
  - `Завершение` (плановая)
  - `Срок` (красным, если просрочен)
  - `Закрыта` (фактическая дата завершения)

### Отличие от «Трудозатраты»

- Блок KPI в этом режиме скрыт.
- Фокус на сроках/планировании, а не на `План/Факт/Δ` в часах.

---

## 5) Представление «Планирование Задачи/день»

### Что решает

- Плановая нагрузка команды на календарной оси.
- Видимость конфликтов и перегруза заранее.

### Правило отбора задач

В planning попадают только задачи, где есть **обе** плановые даты:
- `START_DATE_PLAN`
- `END_DATE_PLAN`

Если одной из дат нет — задача не попадает в planning-матрицу.

### Что выводится

- Слева: выбранный отдел + дочерние отделы 1-го уровня, внутри сотрудники
- Справа: календарная сетка
  - масштабы: дни / недели / месяцы
  - горизонтальный скролл
  - вертикальная линия «сегодня»
- Ячейка содержит интенсивность загрузки и список задач в tooltip
  - в tooltip: задача, дата начала, дата завершения

---

## 6) Скрытое представление «Тепловая карта (факт)»

Режим доступен только при URL-параметре:
- `ws_heatmap=Y`

Если параметр не передан:
- представление не показывается в селекторе.

### Что решает

- Утренний контроль загрузки отдела по фактическим часам.
- Быстрый список «перегружены / в норме / есть ресурс».

### Внутри режима

- Режимы агрегации:
  - Сотрудники
  - Отделы
  - Проекты
- Масштаб: дни / недели / месяцы
- Tooltip по ячейке с задачами периода.

---

## 7) Основные данные, используемые в отчётах

- `b_tasks_elapsed_time` — фактические часы (трудозатраты)
- `b_tasks` — статус, сроки, плановые и фактические даты
- `b_tasks_workgroup` / `b_tasks.GROUP_ID` — привязка к проекту
- `b_sonet_group` — названия проектов
- структура подразделений и пользователи — для орг-иерархии

---

## 8) Что важно помнить бизнесу

- Один и тот же фильтр одинаково влияет на все представления.
- Представления — это разные «линзы» на одни и те же данные:
  - Трудозатраты = факт часов
  - Занятость = сроки и календарь задач
  - Планирование = будущее по плановым датам
  - Heatmap (скрытый) = оперативная фактическая загрузка
